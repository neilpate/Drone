# Research

External references gathered while the project is in flight: datasheets, app notes, vendor documentation, useful blog posts, comparative studies. The goal is to keep the "where did I read that?" question answerable months later.

## What goes here

- **Links** to external resources — datasheet URLs, articles, videos, forum threads, talks.
- **Short notes** summarising what was useful from each (so future-self doesn't have to re-skim a 200-page PDF).
- **Side-by-side comparisons** that informed a decision but don't belong in the ADR itself (the ADR records *what* was chosen and *why*, this folder backs it up with *what was considered*).
- **Local copies** only when the source is at genuine risk of disappearing (small forum posts, deleted blog posts via web archive). Prefer links; check `archive.org` if a link rots.

## What does NOT go here

- **Architecture Decision Records** — those live in [decisions/](decisions/).
- **How-to / build guides** — those become numbered docs (e.g. [03-hardware.md](03-hardware.md) when written).
- **Datasheets in PDF form, copy-pasted.** Link to the vendor URL; if the vendor pulls it, *then* archive a copy with a note about the source. Don't commit copyrighted material wholesale.

## Naming

`topic-short-description.md`, all lowercase, hyphenated. Examples:

- `icm42688-app-notes.md`
- `dshot-protocol-references.md`
- `mahony-vs-madgwick.md`

One topic per file. If a file grows past a few screens of references, split it.

## Status

Empty. Files appear here as research accumulates organically — there is no obligation to pre-populate it.
